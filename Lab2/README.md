## Перетворення ER-діаграми у реляційну схему PostgreSQL


### Метою даної лабораторної роботи є перетворення ER-діаграми предметної області у реляційну схему та її реалізація у СУБД PostgreSQL.

### База даних призначена для автоматизації обліку бібліотечного фонду університету та управління процесом видачі літератури студентам.

---

**Система дозволяє:**

* вести каталог книг та їх авторів,
* зберігати інформацію про студентів (читачів),
* фіксувати факт видачі книги конкретному студенту,
* відстежувати історію та терміни повернення літератури.

---

**Реляційна схема бази даних**

* Author(author_id, name)
* Student(student_card_id, full_name, faculty, email)
* Book(book_id, title, category, publish_year, author_id)
* Loan(loan_id, student_card_id, book_id, issue_date, return_date)

**Основні звʼязки між таблицями**

Один Author може написати багато Book (1:N).
* Один Student може мати багато записів про позичені книги у таблиці Loan (1:N).
* Одна Book може мати багато записів в історії видач у таблиці Loan (1:N).
* Зв'язок багато-до-багатьох (M:N) між Student та Book реалізовано через проміжну (асоціативну) таблицю Loan.
* Усі звʼязки реалізовані за допомогою зовнішніх ключів (FOREIGN KEY) з правилом каскадного видалення (ON DELETE CASCADE).

---

### Обмеження цілісності (Constraints)

**Для підвищення цілісності даних у PostgreSQL та дотримання бізнес-правил використано наступні обмеження:**

UNIQUE: для електронної пошти студента (email), щоб уникнути реєстрації різних читачів на одну адресу.
CHECK: для перевірки коректності року видання книги (publish_year > 0) та перевірки логіки дат (return_date >= issue_date), що унеможливлює повернення книги раніше дати її видачі.
DEFAULT: для автоматичного встановлення поточної дати (CURRENT_DATE) при створенні нового запису про видачу.
